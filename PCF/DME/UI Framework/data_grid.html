DATA GRID

DATA DISPLAY AND NAVIGATION

During display of a data grid of drawing objects, there are two checks to make to determine if a user has the required privileges to view the data objects:  The view of the data via the level; and also the security markings applied to the objects.  Display and navigation of data objects in the drawing shall be determined via the following rules:


1.     Determine if object is at a maturity level that is viewable by the user.


2.     Determine if any objects have a data sensitivity flag set.  If a data sensitivity flag is set on an object, determine if the user has the proper privilege to view the

        data (reference 02_01_00 Account Management)


3.     If it is determined that a user does not have the proper privilege to view the data ( e.g. because of level or the sensitivity markings indicated), disable the

        data grid row from being selected (when the user clicks or double-clicks on the row nothing happens) and mark the row with a lock icon.


4.     If the user does have proper privileges to view the object, select the highest object in the first tab of the grid list.


        a.     When there is only one object in the grid list, select that one and display the tab for the next-lower (child) object. 


        b.     Continue in this fashion until the object’s child is a  detail object (e.g. Equipment, Wire, etc).


        c.     For instance, if you open a drawing that only has one viewable  arrangement, that arrangement should be selected and the Edition tab displayed with

                the highest Edition selected.  If that Edition is the only viewable Edition, then display the first Detail tab and select the highest viewable detail.

FEATURES

The following features should be available on every data grid:


1.    Query: Data Grids are generally populated with a complete set of data, rather than just a subset (e.g. all wires in a bundle, rather than just the first 100

        wires), so no re-query of the database is needed for scrolling down in the data.

4.     Sorting: Sorting of data in a column is supported by clicking on the top of the column. Changing the sort order does not cause a re-query of the database.

        The sort will toggle between ascending or descending.


5.     Find: Find or Search of data in a column is supported by filtering. The Find does not cause a re-query of the database. The Find criteria supports wildcards.


6.     Column Order: Column reorder is supported by the Infragistics data grid.


        a.     User can reorder and resize columns by dragging the column header to reorder, and the column border to resize.


        b.     User can Hide columns.


        c.     User can save setting of “Column Order” for the particular data sheet by using the Columns/Sort buttom show below and described in section X.X of

                this document.


        d.     User can reset to Default settings by using the Columns/Sort button described in section X.X of this document


        e.     The saved Column  order, width, and hidden settings are used whenever that datasheet is opened in the form (tab), in the current session, or future

                sessions for the user.


        f.     User can share the “Column Order” settings with another user through the user preferences file.


7.     Scrolling: Scroll bars will automatically appear if the data grid is too wide or too tall to fit on the screen.


8.     Wide fields:  the data grid with display the first line only up to a set width.  The user can easily display the full text value by doing a right-mouse click and

        selecting View Cell Contents.


9.     Refresh button: Provide a Refresh button on data grid forms. The button will force a re-query against the database, and refill the grid with current

         information.


10.   Double-click: Double clicking on any row will open up a new tab with the detail view of the item in the row just clicked.


11.   Export: Every datagrid will support exporting the contents of the entire data grid to a Microsoft Excel spreadsheet.  This spreadsheet will NOT be formatted

        to allow direct import of the data back into the tool. Importable exports are addressed in 04_17_00_NBE_ADCN-Export and 06_07_00_Drawing_ADCN


12.   Copy: Every datagrid will support copy (ctrl-C and right-mouse-click+Copy) of selected row.


13.   Rows: Use alternating row color to improve readability


14.   Right Click: A right mouse click on a cell provides a list of options: View, View Cell Contents… and Copy


        a.     View Cell Contents…: View Contents shows the content of the cell in a message pop-up.


        b.     Copy:  The copy selection copies the value in the cell to the clipboard so that it can be pasted in a later action later.

Filter

Description

Equals

Rows where the effectivity value exactly matches the input are displayed.  In order for results to be displayed, a valid single effectivity or range must be entered as as the filter criteria.

Does Not Equal

Rows where the effectivity value does not equal the input value are displayed.  In order for results to be displayed, a valid single effectivity or range must be entered as as the filter criteria.

Contains

Rows where the effectivity value has one or more of the input effectivity are displayed.  In order for results to be displayed, a valid single effectivity or range must be entered as as the filter criteria.

Does Not Contain

Rows where the effectivity value does not have any of the input effectivity are displayed.  In order for results to be displayed, a valid single effectivity or range must be entered as as the filter criteria.

Starts With

Rows where the effectivity value starts with the input effectivity are displayed

Ends With

Rows where the effectivity value ends with the input effectivity are displayed
