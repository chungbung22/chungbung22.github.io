SYSTEM ARCHITECTURE

The UI is partitioned into the following major pieces:


a)     A single common UI for the design and manufacturing engineering functionality (BWS Tier 1 architecture)


The basic UI paradigm is a tab-based interface with a drawing tree for navigation.


The essential UI features are (reference graphic image below):


1. Tab-based main window. Major UI windows open in another tab.

       

        a.     The main window shall be designed to a minimum viewing area of 1024 x 768 pixels or about 960 x 768 pixels accounting for scroll bars.

        b.     The user may resize the main window to take up less or more of their screen.

        c.     Resizing of the main window shall not distort or otherwise make any portion of a tab or form unviewable.

        d.     The form layout will be responsive to allow viewing of all content on the page


2.     An expandable drawing tree. Refer to DRAWING TREE.


3.     Clicking on a drawing-tree element opens a window for the selected object. That window opens as a new tabbed entry.


4.     Drawing-tree capabilities:


        a.     The drawing tree can be collapsed to the left side. It can be left there in auto-hide mode, or it can be opened up again permanently.

        b.     In auto-hide mode, it automatically opens when a user hovers over it with the mouse.

        c.     It can be moved and docked to either the left or the right side.     

        d.     It can be pulled into a stand-alone floating window, and then later redocked.


5.     Tab capabilities:


        a.     The tabs can be rearranged, or closed individually.

        b.     They can be pulled into a stand-alone floating window (Refer to TITLE LINE & TEARAWAY TABS for additional information). 

        c.     The user can create additional tab groups by splitting the window horizontally or vertically, and then moving tabs between tab groups.

        d.     Each tab should be labeled with the object type, e.g. “EQ Drawing D12345”.


6.     A main-menu bar at the top of the screen serves the function of standard Windows menus. Refer to specification 02_04_00_Main Menu.


        a.     There is a set of standard menus for File, Edit, View, Tools, and Help. .  Edit not to be shown where not useful, such as main landing page

        b.     There is also a set of application domain specific menus for the main data domains of Systems, Equipment, and Utilities 

        c.     Main menu bar to be white background with black text, standard Windows format.


7.     Datasheets:. The features of the NWS grid are detailed in DATA GRID.


8.     Toolbars: Most datasheets contain a toolbar with actions specific for the data being displayed. (Refer to TOOLBAR for more info)


        a.     Toolbars are always the second row on the form, underneath the title row.

        b.     Each toolbar entry consists of an icon and a 1 or 2 word description.

        c.     The standard set of toolbars can be found in section 3.13 of this document.

        d.     A tool-tip is activated when the mouse hovers over the selection. Refer to ERROR PREVENTION for information on tool-tips


9.     Log: An action and error log at the bottom of the form records significant events, such as object creation and editing.


        a.     The log scrolls.

        b.     The log is for the duration of a single session – it is not persistent. (Note that we may want to write the log to a file).

        c.     The log is collapsible and defaulted to collapsed


10.     Functional Menus: The Main tab contains functional menus for navigating to data in DME


         a.     A functional menu for each drawing type that the user has access to and contains links to the drawing summary, Query and Compare forms.

         b.     A ‘Utilities’ function has links to all tools and utilities that are not specific to a single drawing type (e.g. change management, reservations, etc.).
