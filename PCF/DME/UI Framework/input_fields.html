INPUT FIELDS

INTRODUCTION

Input fields are used to collect user input. They can have multiple states: active, hover, disabled, which can have different background, text, and border colors. Use consistent spacing between rows of fields, and between the field and label text. Mark required fields with a red asterisk to the right of the field label.


TEXT BOXES

• Text should be left-aligned, height should be 1 row (8px), and
    width should be dependent on max char length


• For text fields, only allow the user to type in the number of

    characters equal to or less than the

    size of the field.


• Placeholder text should contain text format (MMDDYY)

PICK LISTS & PULL DOWNS

Use pick lists and pull downs whenever possible to reduce reliance on user memory.


In some cases a value not contained in the list can be entered by the user into the field.  This value will not be added to the list unless the list is specifically set up to incorporate user added entries.


Pick Lists should be used when multiple options can be selected, Pull down should be use when only one option can be selected.


On data-entry fields that are driven by a value list, the value will auto fill as the user types in a value.

RADIO BUTTONS

Use Radio Buttons when a user has to choose only one of several values


Radio buttons should be lined up either in a single row or in a column


Labels for Radio buttons should be displayed to the right of the button.

CHECK BOXES

Use Check Boxes when a user can select one or more of several values


Check Boxes should be lined up either in a single row or in a column


Labels for Check Boxes should be displayed to the right of the check box.
