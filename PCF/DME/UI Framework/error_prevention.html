ERROR PREVENTION

INTRODUCTION

There are multiple ways to provide visual feedback to the user to avoid user error. This includes a pop-up with a message/question or an inline error message. The pop-up is triggered when a button has been clicked, while the inline validation occurs when the user has tabbed out of a field.

INLINE INPUT VALIDATION

Inline error messages provide in-context feedback in response to user input errors. They indicate where the error occurred by highlighting the field in question. The visual feedback will appear once the user has tabbed out of the current field, providing real-time validation, as opposed to a pop-up, which doesn't provide validation until after user input has been submitted.


A highlighted field will consist of a red border, and a error message will be displayed at the top of the form, below the titleline. Syntax should match the case-specific pop up messages, to ensure consistency in the experience.

TOOLTIPS

1.     Tooltips are required for each field label and each button that performs any sort of action. Flyover ToolTip help for each field and button.


2.     Flyover ToolTip for buttons should briefly describe what the button does without just being a repeat of the button label.


3.     Flyover ToolTips for fields should be a brief description of the purpose of the field, without simply restating the name of the field.


4.     Use extended tooltips (information icon with hover/click popup) for areas where users may have more difficulty using or understanding.


5.     Tooltip should appear above the label, button, or input field upon hover
