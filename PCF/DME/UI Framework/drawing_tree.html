DRAWING TREE

INTRODUCTION

The drawing tree is part of the main menu, allowing the user to navigate to spefic data objects.  Here is a brief overview of the guidelines for drawing tree.  For specific details, refer to specification 02_04_00_Main_Menu

BASIC RULES

1.     The drawing tree is a primary navigation mechanism for the UI.


2.     The drawing tree is a standard dockable object.  It can be docked left or right.  It can also float in a separate window apart

        from and outside of the main application window.


3.     The drawing tree is built dynamically as a user clicks on a folder or object.


4.     The drawing tree has folders for every configuration control container: Equipment Drawings, Systems, Change, Standard

        Queries, etc.


5.     The drawing tree has two special objects: Recent and Favorites.


6.     The drawing tree can be collapsed to the left side. It can be left there in auto-hide mode, or it can be opened up again

        permanently.


7.     In auto-hide mode, it automatically opens when a user hovers over it with the mouse.


RECENT

1.     Recent records the list of recently used items.  Every time a dash or detail form is opened, it is recorded in recent as the

        most recent item.


2.     Recent remembers the last twelve items.  Only one occurrence of each item is recorded (no duplicates).

        Clicking on a recent item opens that item in a tab, if the item is not already open.  If it is already open, it is made the active

        tab and the data is refreshed.


3.     The ‘recent’ list is also available from View -> Drawing Tree.


4.     Recent items are automatically qualified by the airplane model, but not by database.


5.     Recent items persist across user sessions on the same machine.


6.     There is no folder or object hierarchy within “Recent”.
