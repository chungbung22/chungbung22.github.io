CODE TEMPLATES

In order to make it easier for developers to re-use existing code and align to standards, we have developed several code templates for custom and MD-UI forms. This allows developers to drag in code snippets with the grid and layout pre-defined, based on the use case.


Each code template will contain a screenshot of its blank state, as well as a populated state with a row height breakdown. The row height breakdown will contain either a pixel count, "Auto" or "*".


• Auto means size to column content

• * means size proportial to grid (displays scrollbar)


To access the code templates from TFS, navigate to the 2CES project, select the NWS folder, followed by the Wiring folder. The  templates will be located in the folder labeled UI Templates



The following are the templates currently available:


MD-UI Form Template

This form is Metadata-driven which means that the UI is generated dynamically and rendered on the fly, as opposed to custom forms. The template should be used for new, edit, view forms such as those found on the detail drawing page.


https://gide-tfs.web.boeing.com/tfs/IT/2CES/_git/NWS.DME.Main?path=%2FNWS%2FWiring%2FUI%20Templates%2FMetadataUIFormTemplate.xaml&version=GB1477533_Create_Templates



Import Form Template

The import form is used when importing graphical data, doing a bulk/mass import, or just doing a simple import. Its structured similar to wizard with a linear flow and steps to follow as you select your import, export to spreadsheet tempalte, and compare data.
