TOOLBAR

INTRODUCTION

Every primary form will have a tool bar at the top of the form that contains icons and menus unique for that form. Below is a sample of  a typical toolbar.

The toolbar will have both text and a graphical icon for each top-level menu choice.


Tool bar items are to be grouped as follows


1.     Current object-specific actions on the left (New, Reserve, Roll…)

2.     Common object functions in the middle (View, Edit…)




WBA Summary

Where there is a drop-down menu, the lower level menu items need not have an icon for each menu choice.


All top-level tool-bar items will have a tool tips.


The tool tips must convey meaningful information which is not simply a duplicate of the label on the menu item. See ERROR PREVENTION for tool tips.


Icons which are never displayed should be hidden.


Icons which are unavailable because of current conditions (such as security restrictions or permissions) should be disabled.


Common icons are pulled from standard set of icons. See ICONS for list.
