PICK LIST & PULL DOWNS

INTRODUCTION

Input fields are used to collect user input. They can have multiple states: active, hover, disabled, which can have different background, text, and border colors. Use consistent spacing between rows of fields, and between the field and label text. Mark required fields with a red asterisk to the right of the field label.


PICK LISTS & PULL DOWNS

• Use pick lists and pull downs whenever possible to reduce
     reliance on user memory.


• In some cases a value not contained in the list can be

     entered by the user into the field.  This value will not be

     added to the list unless the list is specifically set up to

     incorporate user added entries.


• Pick Lists should be used when multiple options can be

     selected, Pull down should be use when only one option can

     be selected.


• On data-entry fields that are driven by a value list, the value

     will auto fill as the user types in a value.
