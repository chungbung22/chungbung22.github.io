PRIMARY FORMS

INTRODUCTION

Primary forms are forms that incorporate multiple actions and at least one grid.


Primary forms should follow the following general layout:


Title

Toolbar

Filter

COMPONENTS

1.     Forms Title Line


The title line is the top line on the form. It includes:


• The correct background color (see Colors).


• Sensitivity Marking appropriate to the region/model or data displayed as appropriate see 02_11_01_Standard_Sensitivity_Marking,and the title of the object being displayed. 


• The Columns/Sort, Refresh and Help action buttons.


 • The title of the form

2.     Toolbar Line


The title line is the top line on the form. It includes:


• The toolbar line is the second line on the form. It consists of the toolbar for all actions on the form. The toolbar is further defined in the TOOLBAR section

3.     Filter Line


The filter line is the third line on the form. It consists of display filters and display options.

Selecting the Purpose. Level, and/or Detail alters the display of the Edition List and Details.  Refer to specification 02_02_01_States_Events_Levels_for_Editions and section DATA GRID for more details.

FEATURES

1.     The user’s identity is validated through Windows login.


2.     Capability to display the Data Dictionary for each screen.


3.     Capability to display the Online Help for each screen.


4.     Allow tab key to move from field to field.


5.     Default text size is 12 point.  The actual size on the User’s monitor is determined by the User’s screen resolution setting.


6.     Unhandled exceptions are logged and managed by the application.  When this type of exception occurs, the following is done:


        a.     A message box is displayed to the user stating that an error has occurred.


        b.     A simple error message is output into the log.  This message only displays the exception type.


        c.     An automated email containing detailed information about the exception and the source is sent to the BWS support distribution list (DL BWS Support) to

               help diagnose the issue.


7.     Closing the application brings up a message, prompting the user to confirm they want to exit.  This message box will not be displayed if the user exits using

        the main menu.


8.     Help will be provided on every form.  Use the following guidelines for incorporating help in to each screen:


        a.     Extended Tooltips can be used on the form to provide guidance where users may have more difficulty using or understanding


        b.     Each form will have a Help button in the Title Bar that will link to the on-line help system topic for that particular form.



SHARING PREFERENCES

During a session a user can change their column settings on various screens, run queries and save to their favorites.  These are referred to as personal preference settings. 


When users save their personal Preferences they are saved in an xml (aka xmi) file in the following location on their personal computer:   C:\Users\(NAME)\AppData\Local\Boeing\BWS_DME\BWS-DME.xmi (Name=nt user account)


This xml file should be available for users to select and send via email, export to a share or save to an exportable medium.  The file can then be imported by other users or the same user while using another machine.


The import/export preferences will be under the Tools Option.
