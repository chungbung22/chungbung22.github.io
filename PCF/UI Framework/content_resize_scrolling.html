CONTENT RESIZE/SCROLLING

3.26 INTRODUCTION

As discussed in the Overview section, the following rules should apply when resizing a window:

        a.     The main window shall be designed to a minimum viewing area of 1024 x 768 pixels or about 960 x 768 pixels accounting for scroll bars.

        b.     The user may resize the main window to take up less or more of their screen.

        c.     Resizing of the main window shall not distort or otherwise make any portion of a tab or form unviewable.

        d.     The form layout will be responsive to allow viewing of all content on the page


To ensure that standards are followed, this section provides additional details on fields, data grids, and button behavior when resizing a window.


1) Buttons should not change in size when resizing a window. Page level buttons that exist at the bottom of the form or window should always be visible, and remain docked at all times. However, buttons that exist in the middle of the form/window do not have to be docked, but should be accessible via scrolling


2) Scrollbars should always appear when the form/window has been resized and all the content can no longer be displayed. A primary and secondary level scrollbar can exist simultaneously (i.e. one for the page level scrolling and one for the data grid)


3) When a form has been shrunk, the data grid should resize accordingly. However, the data grid should always be a minimum of one row height tall, so that the user can still scroll through the grid.


4) When resizing horizontally, the data grid may shrink to fit the contents of the window, but column headers and dropdown content should always be visible. A horizontal scrollbar will appear so that column headers and dropdown content can remain accessible


5) To ensure that dropdown content is always visible and accessible in a data grid, size column width based on the content in the dropdown as opposed to the length of the column headers
