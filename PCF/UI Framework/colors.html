COLORS

3.12 INTRODUCTION

Avoid using color alone as a visual indicator – this could cause end users with visual impairments such as color blindness to be unaware of important

aspects of the user interface.


Color should be combined with other visual techniques such as variation in color value (lighter or darker), icons, or changes in typeface weight to signify changes or differences among items.


Forms that are specific to a drawing should have the color for that drawing in their title area.  Multi-Drawing or non-drawing specific forms should use the gray color gradient shown below.  Below are the colors that depict the different drawings:

3.12.1 ACCESSIBILITY

Users should also be able to easily distinguish the foreground from its background on essential screen elements (e.g., text and icons). Use a color contrast checker to ensure that the color contrast, at a minimum, meets W3C Web Content Accessibility Guidelines (WCAG) 2.0 AA requirements. This will ensure that the colors used in the system are distinguishable to the majority of end users. Contrast checker: http://snook.ca/technical/colour_contrast/colour.html


Avoid using too much color on the page. Strive to use no more than 3 contrasting colors on a page to avoid cognitive overload.


Avoid dark text on dark background or light text on light background. Always ensure that the text is easily visible, and if possible, test on multiple screens to account for screen contrast.
