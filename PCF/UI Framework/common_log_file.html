COMMON LOG FILE

3.8 DEFINITION AND CALLABLE ROUTINE

Processes in BWS routinely generate log files to record actions that happened.  When creating a log file, follow these guidelines.


-A common  callable routine  is used to create the log files.


-Create both a “csv”  and   a  “txt”  file for most large processes.


-The txt file contains a short summary of the results for the process.


-Contains:   Who, when, what process, number of fails, number of success, etc.


-The “csv” contains all the detailed actions for the process.


-For small utilities the single “txt” log file is acceptable.


Location of file:

-Log files are stored automatically during processing.


-Default Location is:  My Documents > BWS > DME Logs 


-When user preference settings are available in DME, use that location for the log files.


File Naming convention:               

                {utility name}_{region id}_{model} _{ year}{month}{date}_{time}_{optional identifier}.{csv,txt}


    Example:     EQMassRelease_Prod1_747_20130621_130201.csv

                         EQImportSpreadsheet_Prod3_777_20130621_030201_C00005.csv

                         EQFindAndReplace_PreProd2_747_20130621_030201.csv

                         EQMassRelease_Prod2_74720130621_130201_PRR123456.txt


DME CSV Format:


Standard Columns:


Date Stamp, Drawing Type, Drawing Key, Dash, Detail Type, Detail Key, Action, Results,  ...


Example:


Date Stamp,  Drawing Type , Drawing, Dash,  Detail Type, Detail Key, Action, Status, Results,  Comment

20130621_030201, EQ Dwg,  C10001, 002,  EQ Detail,  C10001-1 , Revise, Success , Qty 01-> 02

20130621_030201, Bundle Dwg,  W1234, 002,  Wire,  2001R , Add, Fail , gauge 12 -> aa ,   Alpha not allowed for field

20130621_030201, EQ Dwg,  C10001, 002,  Signal, Power , Add, Success ,

20130621_030201, EQ Dwg,  C10001, 002,  Dash,  002 , Add, Success ,

Who:  when,  process name, request criteria, number of failures, number of success, items processed, etc.
