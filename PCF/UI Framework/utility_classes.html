DEVELOPER-LEVEL SPECIFICATIONS

4.1 UTILITY CLASSES

The utility classes are generic, easy to use classes that each serve a general and specific purpose. These utilities are made with the intention that they will be needed by various parts of the application.

1.)     AppUtility


        The AppUtility class contains simple utilities that may be applied “application-wide”.


2.)     CoreUtility


        The CoreUtility class provides a set of routines for marshalling data back and forth between the UI and the “Core” backend system. CoreUtility is located in
        the BWS_Base assembly.


3.)     DataGridUtility


         The DataGridUtility class provides a set of routines that can manipulate data grids. For example, this class can select records and hide columns in the grid.


4.)     FormControlUtility


         The FormControlUtility class provides a generic set of form utilities; nothing dealing with metadata or data in general should be in this class.


5.)     LocalLogUtility


         This class allows developers to get log files using active directory information. Additionally, it can write logs as well.


6.)     MetadataControlUtility


         The MetadataControlUtility allows the caller to populate form controls with “attribute set” data . Additionally, the class allows the developer to extract user

         input from form controls into metadata “attribute sets”.


7.)     TableConverter


          The TableConverter class allows the developer to convert datatables to the Core 2 datatype (DynTable) and vice versa.
