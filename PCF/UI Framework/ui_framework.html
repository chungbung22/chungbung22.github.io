UI FRAMEWORK

3.3 FORM FRAMEWORK

The overall UI pattern follows the NWS pattern of Drawing (base bundle) -> Arrangement (dash/dot)->Detail.  The basic form pattern is thus:


1.     Drawing (e.g. EEC Drawing Summary) - This is a datasheet list of all collectors in a collector group, e.g. all EEC Drawings (Blocks), or all Bundles.


2.     Arrangement (Dash/Dot) List, (e.g. all EEC Drawing dashes for a single EEC Drawing, or all bundle dashes for a single bundle) - This is a datasheet list of

        all arrangements for a single drawing, i.e. all dashes for a single drawing. The arrangement is usually a dash of a drawing. 


3.     Edition List (ADCN/Version)  (e.g. detail for a single EEC-dash/dot) -  This is a datasheet list of all ADCNs for the Arrangement. It incorporates a tabbed

        interface that includes tabs for each detail type in that Arrangement. Each tab is a datasheet list of the detail type.


4.     Detail Tab - Each detail type for the arrangement has a tab,  Each tab lists the attributes of that detail in Grid form wich is available to open in Detail View.      


5.     Detail of Detail (e.g. details for a Equipment, Singal Termination) - This is a single screen that displays all the attributes of a single detail record.  The screen

        is also used to edit that detail.

3.4 NAVIGATION FRAMEWORK

Detail is the lowest level  of the drawing types.  See the 01_03_06_01_BDSD_Drawing for the list of Details for each Edition.   After the edition is created the Detail dropdown will contain the detail types associated with that Edition.  After the user selects a detail in the Detail dropdown; the tabs for the specified detail are displayed on the screen.

The UI presents a uniform and standard way to navigate down the ‘drawing tree’ hierarchy:


1.     Via the drawing tree.


2.     Via the main menu or the top-level menu.


3.     Via double-clicking on a row in a datasheet (takes you to the next level down in the drawing-arrangement hierarchy).


4.     Via starting with an empty ‘Drawing’ form, and directly typing in the name of the drawing (e.g. the bundle number or equipment drawing).  The name is typed

        into a ‘drop down’ box which will auto-fill as the user types.


3.4.1 RULES

1.     A double click on any row of a Drawing Summary datasheet (e.g. EQ Drawing, System, or Wire Bundle) takes you to a form for that single Drawing, with the

        list of Arrangements (dashes/dots) for the drawing, e.g. the list of dashes for a bundle. From the Wire Bundle Summary, a double click on a wire bundle

        drawing row brings up the Wire Bundle Dash form, with a list of dashes for the bundle.


2.     A double click on any row of the arrangement (dash/dot) datasheet (e.g. dashes in a wire bundle) takes you to a form showing the properties of the

        arrangementFrom the Wire Bundle Dash summary, a double click on a wire bundle dash row brings up the Wire Bundle Dash properties tab.


3.     The Drawing form contains tabs for all the arrangement, edtition, and detail record types.  Selecting an edition sets the edition filter for that edition.


4.     Selecting a detail tab (such as the Wire tab) retrieves all the detail data for the Arrangement (dash/dot), at the selected Edition.


5.     For all hierarchy navigation, if the object is already open, its tab is made the active tab and the data on the form is refreshed.
